Multi-Agent Banking Pipeline

Three cooperating agents validate, risk-score, and clear raw bank transactions through a file-based message protocol — built end to end by four meta-agents: a spec writer, a code generator grounded in live context7 lookups, a test generator gated by a coverage hook, and a documentation agent.

validator ──▶ fraud ──▶ compliance

01 · Architecture

Three agents, one file-based protocol

Every hop is a JSON message on disk — no direct function calls between agents, no shared process.

AGENT 1

transaction_validator

Required fields present, amount parses as a positive Decimal, currency is valid ISO 4217. Rejects before fraud detection ever sees it.

AGENT 2

fraud_detector

Weighs high-value (0.8), off-hours (0.3), cross-border (0.3), wire-transfer (0.1) signals into a 0–1 risk score. >0.7 → flagged.

AGENT 3

compliance_checker

Final disposition: rejects unknown transaction types, flags cross-border or fraud-flagged transactions, approves the rest.
